1. Event Propagation в JavaScript описывает механизм распространения событий в дереве DOM. Процесс включает три фазы: фазу захвата (событие распространяется от корневого элемента к целевому), целевую фазу (событие достигает целевого элемента) и фазу всплытия (событие поднимается от целевого элемента к корню). На практике эти знания применяются для реализации делегирования событий, эффективно обрабатывая действия на динамических элементах, а также для управления потоком событий.
2. Promise - объект для обработки асинхронных операций. Работает в 3-х состояниях: ожидание, успешное выполнение, отклонено. Промисы структурируют асинхронный код, позволяя избежать ада колбэков. Альтернативными способами обработки асинхронности являются использование синтаксиса async/await или колбэки. Событийный цикл = координирует выполнение кода, микрозадач и макрозадач, обеспечивая неблокирующую обработку асинхронных операций в однопоточном окружении.
3. ООП - подход к разработке, основанный на объектах. Основные принципы: 1. Инкапсуляция: Сокрытие внутренней реализации объекта, доступ к данным через методы. В js реализуется при помощи замыканий, используя приватные переменные 2. Наследование: Возможность создавать новые классы на основе существующих, наследуя их свойства. JS использует прототипы, позволяя создавать и расширять копии родительского элемента, не меняя его 3. Полиморфизм: Способность объектов разных классов реагировать на один и тот же вызов по-разному. Используя наследование, дочерние классы переопределяют методы родительского класса, предоставляя свою реализацию. Клиентский код работает с общим интерфейсом, не зная конкретного типа объекта.  4. Абстракция: Выделение только существенных характеристик объекта. В JavaScript ООП реализовано на основе прототипной модели, где объекты могут наследовать свойства и методы от других объектов через цепочку прототипов.

Обработка URL браузером включает последовательные этапы: парсинг URL, DNS-запрос для получения IP-адреса, установка TCP-соединения (с дополнительным TLS-рукопожатием для HTTPS), отправка HTTP-запроса, получение и анализ ответа сервера, построение DOM и рендеринг страницы. Для ускорения процесса применяются технологии кэширования (на уровне DNS, HTTP, ресурсов), предварительного соединения (preconnect) и предзагрузки критических ресурсов. Проблемы включают угрозы безопасности (межсайтовый скриптинг, инъекции) и ограничения междоменного взаимодействия, регулируемые политикой CORS и механизмами защиты браузера.
